# Notes on AOT Code Compilation Code (2026)

## 0. Overview

What to expect with the upcoming JEP Ahead-of-Time Code Compilation.

This is last updated around 2026/05/26. It may be out-of-date by the time you read this doc.

## 1. Setup

- Get the draft PR from https://github.com/openjdk/jdk/pull/30778

  Note: a new PR may be created when the JEP is integrated into the JDK mainline. 
  Please look for it first.

- You need hsdis to disassemble the code generated by the HotSpot JIT and/or AOT compiler. Google
  for "build hsdis-amd64.so", etc. If you are using Linux/x64, see https://medium.com/ayte-io/building-hsdis-amd64-so-hsdis-i386-so-for-linux-ec51593a9318

- On linux, you need to `export LD_LIBRARY_PATH=/path/to/hsdis`


## 2. Inspect JIT compiled method

Use the following program. Note the use of numbers like `0x7a7a7a` to make it easier to search inside the assembly code.

```
public class IntLoop {
    int a;
    int counter;
    public static void main(String args[]) {
        for (int i = 0; i < 2; i++) {
            (new IntLoop()).doit();
        }
    }

    void doit() {
        counter = 0;
        for (int i = 0; i < 0x10000; i++) {
            test(a);
        }
        System.out.format("counter = 0x%x\n", counter);
    }

    void func() {
        counter += 1;
    }

    void test(int m) {
        counter += m * 0x2a2a2a;
        for (int i = 0; i < 0x7a7a7a; i++) { // Should be unrolled
            func();                          // Should be inlined
        }
    }
}
```

Build it and see how the JIT compiles the loop in `test1()`:

```
$ rm -rf tmpclasses
$ mkdir tmpclasses
$ javac -d tmpclasses IntLoop.java
$ jar cf IntLoop.jar -C tmpclasses .

$ time java -cp IntLoop.jar IntLoop
counter = 0x7a7a0000
counter = 0x7a7a0000

real	0m0.034s
user	0m0.022s
sys 	0m0.017s

$ java -XX:+UnlockDiagnosticVMOptions '-XX:CompileCommand=print,IntLoop::*' -cp IntLoop.jar IntLoop > log.txt
```

You can see the JIT has reduced the loop into a constant "add". My comments are marked with `;;` in the assembly listing.

## 2. Inspect AOT compiled method


First, do a training run and record the execution profile

```
$ java -cp IntLoop.jar -XX:AOTMode=record -XX:AOTConfiguration=app.config IntLoop 
counter = 0x7a7a0000
counter = 0x7a7a0000
 AOTConfiguration recorded: app.config
```

Then, build the AOT cache

```
java -cp IntLoop.jar -XX:AOTMode=create -XX:AOTConfiguration=app.config -XX:AOTCache=app.aot -XX:+UnlockDiagnosticVMOptions '-XX:CompileCommand=print,IntLoop::*' > log.txt
```

Start time is improved:  0m0.034s -> 0m0.013s

```
$ time java -cp IntLoop.jar -XX:AOTCache=app.aot IntLoop
counter = 0x7a7a0000
counter = 0x7a7a0000

real	0m0.013s
user	0m0.011s
sys 	0m0.009s
```

Compared to mainline JDK (AOT cache, without AOT-compiled methods)

```
real	0m0.018s
user	0m0.015s
sys 	0m0.012s
```

--> Faster start-up, less system time due to fewer JIT compilation (note: unscientific measurement)
